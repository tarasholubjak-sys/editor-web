# Стандарт оформлення Selfy Knowledge Base

## 1. Мова

- **Українська**. Тільки.
- Якщо вхідний текст російською/англійською — **переписати українською**, зберігши зміст.
- Без жаргону "по-замовчуванням" / "юзер" / "тікет" — використовуй українські еквіваленти ("за замовчуванням" / "користувач" / "звернення").

## 2. Стиль викладу

- **Короткі речення.** Якщо речення > 25 слів — розбий.
- **Прості абзаци.** Один абзац = одна думка.
- **Активний стан.** Не "клієнтом має бути надіслано" — а "клієнт надсилає".
- **Конкретно.** Не "потрібно діяти швидко" — а "відправити протягом 3 днів".
- **Без води.** Викидай: "як ви знаєте", "очевидно", "хочу зазначити", "слід пам'ятати", "важливо розуміти".

## 3. Структура

### 3.1 H1 — назва документа

- Одна на документ.
- Без префіксів типу `[ІНСТРУКЦІЯ]`, без емодзі.
- Точно описує тему: ✅ "Алгоритм роботи з поверненнями" / ❌ "Повернення".

### 3.2 H2 — основні розділи

- 5-12 на документ.
- **Один емодзі на початку** для візуальної навігації.
- Без термінів-питань: ✅ "🎯 Мета" / ❌ "🎯 Яка мета цього документа?".

### 3.3 H3 — підрозділи

- Тільки коли реально треба розбити H2 на частини.
- Без емодзі.
- ⚠️ H4 не показується в Outline TOC — уникай.

### 3.4 Стандартні смайлики для H2

| Розділ | Смайлик |
|---|---|
| Мета | 🎯 |
| Для кого | 👤 |
| Коротко (TL;DR) | 🧭 |
| Терміни / визначення | 🧩 |
| Алгоритм / Кроки | ✅ |
| Правила / Політики | 📌 |
| Хто платить / Відповідальність | 💰 |
| Терміни (час) | ⏱ |
| Інструменти / Доступи | 🧰 |
| Приклади / Кейси | 🧪 |
| Шаблони фраз | 💬 |
| Типові помилки | ⚠️ |
| Контроль якості / Перевірка | 🔎 |
| Чек-лист | ☑️ |
| Якщо щось пішло не так | 🆘 |
| Шаблони / Заготовки | 📝 |
| Потрібно уточнити | ⚠️ |
| Посилання / Матеріали | 📎 |

## 4. Шапка документа

Завжди на початку, ВІДРАЗУ під H1:

```markdown
> **Власник процесу:** [ПІБ / роль] *(або "потребує уточнення")*
> **Для кого:** [відділ / ролі]
> **Оновлено:** YYYY-MM-DD
> **Статус:** Актуально *(або: Чернетка / Потребує ревізії)*
> **Теги:** #тег1 #тег2 #тег3
```

Тегі — 3-8, маленькими літерами, через `_` для багатослівних:
- ✅ `#повернення #брак #waldi #1с #легке_повернення`
- ❌ `#Powernennya #ProductReturn`

## 5. Чек-листи

Завжди `- [ ]` для дій:

```markdown
- [ ] Отримав фото + відео
- [ ] Зафіксував номер замовлення
- [ ] Заповнив файл в 1С
```

❌ Не використовуй `* ` або `1. ` для чек-листів — це не чек-листи, це списки.

## 6. Таблиці

Обов'язково з заголовком + вирівнюванням:

```markdown
| Колонка 1 | Колонка 2 |
|---|---|
| Дані | Дані |
```

Використовуй ТАМ, де є:
- Порівняння (брендів, тарифів, варіантів)
- Розгалуження "якщо X → то Y"
- Список параметрів (терміни / відповідальні / суми)

## 7. Цитати / Шаблони фраз

Для готових формулювань ВЖЕ форматуй з `>`:

```markdown
> "Добрий день! Це [Ім'я] з компанії Selfy..."
```

Або з підблоком категорії:

```markdown
**Вхід:**
> "Добрий день! Це [Ім'я] з компанії Selfy."

**Кваліфікація:**
> "У вас магазин чи інтернет-магазин?"
```

## 8. Notice блоки

Дотримуйся семантики:

| Тип | Коли |
|---|---|
| `:::info` | Довідкова інформація, факт |
| `:::warning` | Ризик, обмеження, дедлайн |
| `:::tip` | Best practice, хитрість |
| `:::success` | Очікуваний результат, критерій ОК |

❌ Не зловживай: 5-7 notice на документ — це **дуже багато**.

## 9. Mermaid діаграми

Використовуй для:
- Алгоритмів з гілками (`flowchart TD`)
- Послідовності взаємодій (`sequenceDiagram`)
- Стейт-машин (`stateDiagram-v2`)

Простота важливіша за деталізацію — 5-10 нод, не більше 15.

## 10. Посилання

### Внутрішні (на інші документи Outline) → Document mentions:

```markdown
[[Алгоритм роботи з поверненнями]]
```

### Зовнішні (на сторонні сайти) → Markdown links:

```markdown
[Назва сторінки](https://example.com)
```

❌ Не вставляй сирі URL без link text.

## 11. Емодзі

- На H2 — **один** на початку.
- В тексті — **дуже економно**. Один-два на абзац максимум.
- ❌ Не "🎯🚀✨ Все буде 💪 круто 🎉🎉🎉".

## 12. Що ЗАВЖДИ має бути

У кожному документі:

1. ✅ Шапка з метаданими
2. ✅ Мета (🎯) — навіщо цей документ
3. ✅ Коротко (🧭) — суть у 3-5 пунктах
4. ✅ Основна частина (за шаблоном типу документа)
5. ✅ Типові помилки (⚠️) — мінімум 3 для процесних SOP
6. ✅ Чек-лист (☑️ / 🔎) — критерії "ОК"
7. ✅ Потрібно уточнити (⚠️) — якщо є дірки
8. ✅ Пов'язані документи (📎) — backlinks

## 13. Що ЗАБОРОНЕНО

- ❌ Блок "Зміст" у тілі (Outline сам)
- ❌ H4 для важливих розділів
- ❌ CAPS LOCK для акцентуації (використовуй `**bold**` або notice)
- ❌ "!!!!" — це не акцентуація
- ❌ Перевантаження емодзі
- ❌ Вигадані цифри/політики/імена
- ❌ Англіцизми ("апрув", "екаунт", "ричардстал")
- ❌ Незрозумілі скорочення без розшифровки
- ❌ Питання "Зрозуміло?" в кінці секції
